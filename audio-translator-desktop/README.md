# 实时音频翻译器 - 桌面应用

基于 Electron 的桌面应用，可以捕获系统音频输出，通过豆包同传 API 实时翻译成中文字幕，并以可拖拽浮窗形式显示。

## 功能特性

- 🎙️ 捕获系统音频输出
- 🌐 集成豆包语音同传 API
- 💬 实时浮窗字幕显示
- 🖱️ 浮窗可自由拖拽移动
- ⚙️ 可视化配置界面
- 🔒 本地运行，数据安全

## 使用前准备

### 1. 获取豆包 API Key

1. 访问 [火山引擎控制台](https://console.volcengine.com/)
2. 注册/登录账号
3. 进入「语音技术」→「实时语音翻译」
4. 创建应用，获取 `App Key` 和 `Resource ID`

### 2. 安装音频捕获驱动（macOS）

macOS 需要安装虚拟音频设备来捕获系统音频：

**推荐使用 BlackHole:**

```bash
brew install blackhole-2ch
```

或者从 [BlackHole 官网](https://github.com/ExistentialAudio/BlackHole) 下载安装。

安装后，在「系统设置」→「声音」→「输出」中选择 "BlackHole 2ch"。

## 安装和运行

### 1. 安装依赖

```bash
cd audio-translator-desktop
npm install
```

### 2. 启动应用

```bash
npm start
```

## 使用说明

### 1. 配置 API

1. 打开应用
2. 在主窗口填入从火山引擎获取的 `App Key` 和 `Resource ID`
3. 选择源语言（如印尼语）和目标语言（简体中文）
4. 点击「💾 保存配置」

### 2. 开始翻译

1. 确保系统音频输出设置为 BlackHole（或其他虚拟音频设备）
2. 点击「▶️ 开始翻译」
3. 播放需要翻译的音频/视频
4. 浮窗会实时显示翻译字幕

### 3. 使用浮窗

- 浮窗默认位于屏幕右下角
- 可拖拽浮窗任意移动位置
- 点击浮窗右上角的 `×` 可隐藏浮窗
- 在主窗口点击「👁️ 显示浮窗」可重新显示

### 4. 停止翻译

点击主窗口的「⏹️ 停止翻译」按钮。

## 技术栈

- **Electron** - 桌面应用框架
- **Web Audio API** - 音频处理
- **豆包同传 API** - 语音翻译
- **WebSocket** - 实时通信

## 注意事项

1. **系统音频设置**：macOS 需要设置音频输出为虚拟音频设备才能捕获系统音频
2. **网络连接**：需要稳定的网络连接调用豆包 API
3. **API 配额**：注意火山引擎的 API 使用配额限制
4. **音频质量**：建议使用 16kHz 采样率的音频以获得最佳翻译效果

## 故障排除

### 浮窗不显示字幕

- 检查 API Key 是否正确配置
- 检查网络连接是否正常
- 查看浏览器控制台的错误信息

### 无法捕获系统音频

- 确认已安装虚拟音频设备
- 检查系统音频输出设置
- 重启应用重试

### API 连接失败

- 确认 API Key 和 Resource ID 正确
- 检查网络连接
- 查看火山引擎控制台是否有配额限制
